### Phase 3: Location Services & Background Tasks

✅ **Subtask 3.1: Create location service** (COMPLETED)
- Created LocationService interface with comprehensive location tracking methods
  - getCurrentLocation() with configurable accuracy levels
  - getLastKnownLocation() for fast cached location retrieval
  - onLocationChanged stream for continuous location updates
  - startLocationUpdates() and stopLocationUpdates() for lifecycle management
  - Permission checking and requesting (checkPermission(), requestPermission())
  - isLocationServiceEnabled() to verify device location services
  - distanceBetween() for calculating distances between coordinates
- Created LocationServiceImpl using geolocator package for battery-efficient tracking
  - Maps accuracy levels: low (~3km), balanced (~100m), high (~10m), best (~0m)
  - Implements distanceFilter to reduce unnecessary location updates
  - Proper permission handling with descriptive error messages
  - Stream-based location updates with configurable distance filter (default 10m)
  - 30-second timeout for location requests to prevent hanging
  - Converts geolocator Position to LocationData format
- Added exception classes: LocationServiceException and LocationPermissionException
- Follows existing service patterns with proper resource disposal
- Includes Riverpod providers: locationServiceImplProvider and locationServiceOverrideProvider
- Commit: 6a0688b
