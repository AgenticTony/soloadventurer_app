# Tag Management System

Comprehensive tagging system for categorizing and organizing journal entries in the Travel Journal feature.

## Overview

The Tag Management System provides a complete solution for creating, managing, and using custom tags to organize journal entries. Tags can have custom colors and icons, making it easy to visually categorize and filter entries.

## Features

- ✅ Create custom tags with names, colors, and icons
- ✅ Edit and delete tags
- ✅ Assign multiple tags to journal entries
- ✅ Tag usage tracking (see how many entries use each tag)
- ✅ Search tags by name
- ✅ Popular tags view (sorted by usage)
- ✅ Tag picker widget for easy selection
- ✅ Full CRUD operations with error handling
- ✅ Material Design 3 UI

## Database Schema

### Tags Table

```sql
CREATE TABLE tags (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL,
  name VARCHAR(100) NOT NULL,
  color VARCHAR(7),              -- Hex color code
  icon VARCHAR(50),              -- Emoji or icon name
  usage_count INTEGER DEFAULT 0,
  created_at TIMESTAMPTZ,
  UNIQUE(user_id, name)
);
```

### Journal Tags Junction Table

```sql
CREATE TABLE journal_tags (
  journal_entry_id UUID NOT NULL,
  tag_id UUID NOT NULL,
  created_at TIMESTAMPTZ,
  PRIMARY KEY (journal_entry_id, tag_id),
  FOREIGN KEY (journal_entry_id) REFERENCES journal_entries(id) ON DELETE CASCADE,
  FOREIGN KEY (tag_id) REFERENCES tags(id) ON DELETE CASCADE
);
```

## Usage Examples

### 1. Displaying All Tags

```dart
class TagListScreenExample extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final tagListState = ref.watch(tagListProvider);

    return Scaffold(
      appBar: AppBar(title: const Text('My Tags')),
      body: tagListState.isLoading
          ? const Center(child: CircularProgressIndicator())
          : ListView.builder(
              itemCount: tagListState.tags.length,
              itemBuilder: (context, index) {
                final tag = tagListState.tags[index];
                return ListTile(
                  leading: Icon(Icons.label, color: _parseColor(tag.color)),
                  title: Text(tag.name),
                  subtitle: Text('${tag.usageCount} entries'),
                );
              },
            ),
    );
  }

  Color _parseColor(String? colorString) {
    if (colorString == null) return Colors.blue;
    return Color(int.parse(colorString.replaceFirst('#', '0xFF')));
  }
}
```

### 2. Creating a New Tag

```dart
final repository = ref.read(tagRepositoryProvider);
final tag = Tag(
  id: '',  // Will be generated by server
  userId: '',
  name: 'Adventure',
  color: '#FF6B6B',
  icon: '🏔️',
  usageCount: 0,
  createdAt: DateTime.now(),
);

final result = await repository.createTag(tag);
result.fold(
  (failure) => print('Error: ${failure.message}'),
  (createdTag) => print('Created tag: ${createdTag.name}'),
);
```

### 3. Using Tag Picker in Journal Entry Creation

```dart
class CreateJournalEntryScreen extends ConsumerStatefulWidget {
```

### 5. Searching Tags

```dart

  (failure) => print('Error: ${failure.message}'),
```

```dart
    }
  },
);
```

## State Management

### TagListProvider

Manages the list of all tags:

```dart
final tagListState = ref.watch(tagListProvider);

// Properties:
// - tags: List of all tags
// - isLoading: Boolean
// - error: String?

// Methods:
ref.read(tagListProvider.notifier).loadTags();
ref.read(tagListProvider.notifier).loadPopularTags(limit: 20);
ref.read(tagListProvider.notifier).searchTags('query');
ref.read(tagListProvider.notifier).clearError();
```

### TagFormProvider

Manages tag creation/editing:

```dart
final formState = ref.watch(tagFormProvider);

// Properties:
// - name: String
// - color: String?
// - icon: String?
// - isLoading: Boolean
// - error: String?
// - tag: Tag? (when editing)

// Methods:
ref.read(tagFormProvider.notifier).updateName('Beach');
ref.read(tagFormProvider.notifier).updateColor('#FFD93D');
ref.read(tagFormProvider.notifier).updateIcon('🏖️');
ref.read(tagFormProvider.notifier).loadTag(tagId);
final success = await ref.read(tagFormProvider.notifier).saveTag();
ref.read(tagFormProvider.notifier).reset();
```

### EntryTagsProvider

Manages tags for a specific journal entry:

```dart
final entryTagsState = ref.watch(entryTagsProvider);

// Properties:
// - tags: List of all available tags
// - selectedTagIds: List of selected tag IDs
// - isLoading: Boolean
// - error: String?

// Methods:
ref.read(entryTagsProvider.notifier).loadEntryTags(entryId);
ref.read(entryTagsProvider.notifier).loadAvailableTags();
ref.read(entryTagsProvider.notifier).toggleTag(tagId);
ref.read(entryTagsProvider.notifier).addTag(tagId);
ref.read(entryTagsProvider.notifier).removeTag(tagId);
final success = await ref.read(entryTagsProvider.notifier).saveEntryTags(entryId);
```

## Integration with Journal Entries

### Step 1: Add Tag Picker to Entry Creation Screen

```dart
// In your journal entry creation screen
final List<String> _selectedTagIds = [];

// Add to form
TagPicker(
  selectedTagIds: _selectedTagIds,
  onTagsChanged: (tagIds) {
    setState(() => _selectedTagIds = tagIds);
  },
),
```

### Step 2: Save Tags When Creating Entry

```dart
final entry = JournalEntry(
  // ... other fields
);

// Create the entry
final createdEntry = await journalRepository.createEntry(entry);

// Then save the tags
if (_selectedTagIds.isNotEmpty) {
  await journalRepository.updateTagsForEntry(
    createdEntry.id,
    _selectedTagIds,
  );
}
```

### Step 3: Display Tags on Entry Detail Screen

```dart
// Load tags for the entry
final tagIds = await journalRepository.getTagsForEntry(entryId);

// Get tag details
final tagRepository = ref.read(tagRepositoryProvider);
final tagsResult = await tagRepository.getTags();

// Display
Wrap(
  spacing: 8,
  runSpacing: 8,
  children: tagIds.map((tagId) {
    final tag = tagsResult.fold((l) => null, (r) => r.firstWhere((t) => t.id == tagId));
    if (tag == null) return const SizedBox();

    return Chip(
      label: Text(tag.name),
      backgroundColor: _parseColor(tag.color)?.withOpacity(0.2),
    );
  }).toList(),
)
```

## Error Handling

All repository methods return `Either<Failure, T>` for proper error handling:

```dart
final result = await repository.createTag(tag);
result.fold(
  (failure) {
    // Handle error
    switch (failure.code) {
      case 'duplicate_tag':
        showError('A tag with this name already exists');
        break;
      default:
        showError('Failed to create tag: ${failure.message}');
    }
  },
  (tag) {
    // Success
    showSuccess('Tag created successfully');
  },
);
```

## Performance Considerations

1. **Tag Loading**: Tags are loaded once and cached in the provider
2. **Usage Count**: The database automatically updates usage_count via triggers
3. **Search**: Search is performed server-side with `ilike` for efficiency
4. **Popular Tags**: Pre-sorted by usage_count for quick display

## Troubleshooting

### Tags not appearing in picker

1. Check that `entryTagsProvider.notifier.loadAvailableTags()` was called
2. Verify tags exist in the database
3. Check for errors in `entryTagsState.error`

### Can't save tags to entry

1. Ensure entry was created first
2. Check that tag IDs are valid
3. Verify database permissions for `journal_tags` table

### Colors not displaying correctly

1. Ensure color string is valid hex format (#RRGGBB)
2. Check color parsing logic in widgets
3. Verify color is saved in database correctly
