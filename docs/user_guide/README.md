## Documentation Style

User guides should:
- Be written in clear, friendly language
- Include practical examples and scenarios
- Provide step-by-step instructions
- Use visual formatting (icons, tables, lists) for readability
- Address common questions and concerns
- Include troubleshooting sections

## Contributing

When adding new user guides:
1. Create a new `.md` file in this directory
2. Use clear, descriptive filenames
3. Follow the existing style and formatting
4. Update this README with a link to your guide
5. Test the guide with actual users if possible
