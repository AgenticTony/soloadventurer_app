## Tips for Effective Updates

1. Be specific about what was completed
2. Include relevant code snippets in documentation
3. Link to relevant PRs or commits
4. Note any deviations from planned approach
5. Document rationale for significant decisions
6. Keep a running list of technical debt
7. Update diagrams and visual aids

## Common Pitfalls to Avoid

1. Forgetting to update interconnected documents
2. Not documenting blockers immediately
3. Skipping documentation for "small" changes
4. Not linking related changes across documents
5. Leaving outdated TODOs in documentation
